# Lanflow 实验室使用说明

> 本文档面向实验室日常使用和维护，不涉及开发细节。

## 一、这是什么？

Lanflow 是我们实验室的**局域网流量监控系统**。它部署在 5090x2 服务器（`192.168.1.108`）上，用来统计每个人通过校园网用了多少流量、访问了哪些网站。

**访问地址：http://192.168.1.108:8080**

## 二、三个功能页面

### 1. 实时监控

打开网页默认就是这个页面。

- **顶部三个卡片**：在线设备数、全网总上行流量、全网总下行流量
- **设备表格**：每个人的电脑名称、IP、当前网速、今日累计流量
- **"详情"按钮**：点击可以看到这个人今天都访问了哪些网站，各用了多少流量
- 数据每 2 秒自动刷新，不用手动刷新页面

### 2. 历史统计

查看过去某一天/某一周/某一月的流量统计。

- 选择时间范围和日期，点"查询"
- **柱状图**：各设备流量对比
- **排行表格**：前三名会有金银铜色标记
- **点击某一行**：展开该设备的详细趋势图和域名流量明细

### 3. 设备管理

给 IP 地址起名字，方便识别谁是谁。

- **已发现的设备**：蓝色标签是没起名的 IP，绿色是已经起好名字的
- 点击标签会自动填入表单，输入名称后点"保存"即可
- 已命名的设备在所有页面都会显示名称而不是 IP

## 四、日常注意事项

### 对你有什么影响？

**几乎没有影响。** 你的电脑正常上网就行，不需要安装任何软件，不需要做任何设置。Lanflow 只是在网关上默默统计流量，不会修改你的数据包，不会影响你的网速。

### 你的隐私

- Lanflow **能看到**：你访问了哪些网站的域名（如 bilibili.com），以及流量大小
- Lanflow **看不到**：你具体看了什么内容（如看了什么视频、发了什么消息）——因为流量都是加密的
- 如果你使用了代理/VPN（如 Clash），Lanflow 只能看到你连接了代理服务器，看不到你通过代理访问了什么

### 校园网流量

我们实验室共享一个校园网账号。校园网可能有流量限制或计费，请注意：
- 避免长时间下载大文件（如数据集、Docker 镜像等），尽量走实验室内网或提前下载
- 如果发现某天流量异常高，可以到 Lanflow 查看是谁用了多少、去了哪些网站
- 大文件下载建议在非高峰时段进行

## 五、常见问题

### Q：网页打不开了？

1. 先检查 5090x2 服务器（192.168.1.108）是否开机
2. 如果服务器在运行但网页打不开，SSH 登录服务器执行：
   ```bash
   sudo systemctl status lanflow    # 查看服务状态
   sudo systemctl restart lanflow   # 重启服务
   ```

### Q：所有人都断网了？

**最可能的原因是 5090x2 服务器关机或崩溃了。**

**快速恢复方法（任选一个）：**

**方法一：重启服务器**
- 到机房按电源键重启 192.168.1.108 这台机器
- 开机后 lanflow 会自动启动，网络自动恢复

**方法二：绕过 Lanflow（服务器修不好时）**
1. 用手机连实验室 WiFi（或直接网线连路由器）
2. 浏览器打开 `http://192.168.1.1`（路由器管理界面）
3. 进入"路由设置" → "DHCP 服务器"
4. 把"网关"从 `192.168.1.108` 改回 `0.0.0.0`
5. 点保存
6. 各电脑断开重连网络（或等几分钟自动恢复）

> 改回后所有人直接通过路由器上网，Lanflow 停止监控。等服务器修好后再改回来。

### Q：新来的同学/新电脑怎么设置？

不需要设置任何东西！直接插网线或连 WiFi 就行。连上后会自动出现在 Lanflow 的监控列表中。

记得到 http://192.168.1.108:8080 的"设备管理"页面给新电脑起个名字。

### Q：我的电脑换了 IP 地址怎么办？

IP 通常不会变（DHCP 会尽量分配相同的 IP）。如果确实变了，到"设备管理"页面给新 IP 重新起名就行。旧 IP 的历史数据不会丢失。

建议在路由器中绑定 MAC-IP（"路由设置" → "IP与MAC绑定"），这样 IP 就不会变了。

### Q：怎么看我今天用了多少流量？

1. 打开 http://192.168.1.108:8080
2. 在"实时监控"页面找到你的设备名称
3. "今日总计"列就是你今天的总流量

### Q：怎么看我的流量都去了哪里？

1. 在"实时监控"页面找到你的设备
2. 点击右边的"详情"按钮
3. 弹出窗口会显示你访问的各个网站及流量占比

### Q：怎么看过去几天的数据？

1. 点击"历史统计"标签
2. 选择时间范围（按天/按周/按月）和日期
3. 点"查询"

### 常用命令

```bash
# 查看服务状态
sudo systemctl status lanflow

# 重启服务（不影响其他人上网）
sudo systemctl restart lanflow

# 查看实时日志
sudo journalctl -u lanflow -f

# 查看 watchdog 日志
tail -f /opt/lanflow/logs/watchdog.log

# 清理流量数据重新统计（保留设备命名）
sudo systemctl stop lanflow
sqlite3 /opt/lanflow/data/lanflow.db "DELETE FROM traffic_stats; DELETE FROM domain_stats; VACUUM;"
sudo systemctl start lanflow
```

### 更新程序

如果需要更新 Lanflow（修 bug 或加功能）：

```bash
# 在开发机上（有 Go 环境）
git clone https://github.com/wyzz973/lanflow.git
cd lanflow
export GOPROXY=https://goproxy.cn,direct
go build -o lanflow ./cmd/lanflow/

# 传到服务器
scp lanflow chenxu@192.168.1.108:/home/chenxu/

# SSH 到服务器替换
ssh chenxu@192.168.1.108
sudo systemctl stop lanflow
sudo cp /home/chenxu/lanflow /opt/lanflow/lanflow
sudo systemctl start lanflow
```


### 网络架构说明

```
[校园网] ←→ [TP-LINK 路由器 192.168.1.1] ←→ [5090x2 服务器 192.168.1.108] ←→ [所有电脑]
                                                    ↑
                                              Lanflow 在这里抓包
                                              Linux 内核负责转发
```

**关键配置点：**

1. **路由器 DHCP**：网关设置为 `192.168.1.108`（让所有设备把流量发给服务器）
2. **服务器静态 IP**：`192.168.1.108`，网关指向 `192.168.1.1`（服务器自己直连路由器）
3. **IP 转发**：`net.ipv4.ip_forward=1`（让服务器转发其他设备的流量）
4. **NAT 规则**：`iptables MASQUERADE`（让转发的流量能正常回来）
5. **DNS 转发**：`iptables DNAT 53 → 223.5.5.5`（让设备的 DNS 查询正常工作）

以上所有配置已持久化，**服务器重启后会自动恢复**。

### 高可用保障

| 层级 | 措施 | 说明 |
|------|------|------|
| 进程 | systemd Restart=always | lanflow 崩溃 3 秒后自动重启 |
| 进程 | OOMScoreAdjust=-900 | 内存不足时优先保护网关进程 |
| 网络 | IP 转发独立于 lanflow | **lanflow 挂了不会断网**，只是暂时没有监控数据 |
| 系统 | watchdog 脚本 | 每分钟检查 IP 转发、iptables、lanflow 服务 |
